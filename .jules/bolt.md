## 2025-02-05 - Filter by distance performance bottleneck
**Learning:** In `ConstrainedSMOTE`, the `_filter_by_distance` method calculated Euclidean distances using an O(N*M) nested Python loop and list comprehensions. Replacing this with a vectorized NumPy batched implementation using `scipy.spatial.distance.cdist` dramatically improves performance (e.g. from 8.7s to 0.3s) and prevents out-of-memory errors. The approach groups samples by label, grabs both original and synthetic embeddings of that label, computes the full pairwise distance matrix via `cdist`, then takes `.min(axis=1)` to find the closest neighbor distance for thresholding.
**Action:** Always favor batched/vectorized `cdist` (from SciPy or PyTorch equivalents) over Python `for` loops when computing distance thresholds or nearest neighbors across large sets of embedding vectors.
