
## 2025-02-13 - PyTorch cdist for Distance Matrix Optimization
**Learning:** In PyTorch codebases doing matrix-heavy analytical operations (like pairwise image comparisons for diversity scoring), translating `cpu().numpy()` conversions + SciPy `pairwise_distances` into `torch.cdist` provides significant ~2-3x speedup. Even larger speedups exist when scaling inputs up. It is important to set `unbiased=False` on `torch.std` to match `np.std` default behavior and match devices for tensor initializations (`torch.randperm`, `torch.triu_indices`) to prevent implicit CPU synchronization.
**Action:** When computing any metric that involves distances across an entire batch or dataset, leverage native PyTorch functions (like `cdist`, `triu_indices`, etc.) to stay on the GPU and prevent transferring memory back and forth to the CPU. Always explicitly ensure `torch.std` uses `unbiased=False` when replacing numpy functions unless unbiased standard deviation is genuinely expected.
