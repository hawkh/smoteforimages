## 2026-05-12 - Native PyTorch Distance Computations for Quality Metrics
**Learning:** Computing pairwise distances in `compute_diversity_metrics` using `sklearn.metrics.pairwise.pairwise_distances` required expensive CPU memory transfers and flattening. Using native PyTorch operations like `torch.cdist` and extracting the upper triangle via `torch.triu_indices` keeps the computation on the original device (potentially leveraging the GPU) and improves CPU execution time by ~50%.
**Action:** Always prefer native PyTorch vectorized operations like `torch.cdist` over cross-library NumPy/Scikit-learn calls when processing intermediate or final PyTorch tensors. Ensure compatibility by keeping PyTorch parameters matching NumPy defaults (e.g., `unbiased=False` for standard deviation).
