## 2024-05-13 - Optimize _filter_by_distance with Vectorization and Batching
**Learning:** The previous implementation of `_filter_by_distance` in `ConstrainedSMOTE` utilized an O(N x M) nested loop for distance thresholding, causing performance bottlenecks and high execution times during sample generation. Replacing it with `scipy.spatial.distance.cdist` provides significant speedups but computing full matrices between generated and source embeddings can cause Out-of-Memory (OOM) errors. We learned that `scipy.spatial.distance.cdist` should be combined with batching (e.g., chunk sizes of 2000 items) to optimally balance GPU/CPU memory and fast, vectorized math operations.
**Action:** When filtering or calculating pairwise distances across large sets of embedding vectors, use vectorized `scipy.spatial.distance.cdist` computations rather than nested Python loops, and enforce chunking (batch processing) to safely avoid memory spikes.
