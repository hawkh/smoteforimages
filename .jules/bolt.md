## 2025-02-12 - PyTorch vs NumPy Pairwise Distances Bottleneck
**Learning:** In `QualityAssessor`, calculating diversity metrics using `sklearn.metrics.pairwise_distances` introduces a significant performance penalty because it requires moving large image tensors (e.g., 1000x3x224x224) from GPU to CPU via `.cpu().numpy()`. This creates a memory transfer bottleneck and misses out on GPU-accelerated pairwise distance computations. Using `torch.cdist` directly on the tensors is about 4x faster on CPU and orders of magnitude faster when tensors are on GPU.
**Action:** Always prefer native PyTorch operations (`torch.cdist`, `torch.triu_indices`) for tensor math instead of moving data to CPU/NumPy, especially for large multidimensional arrays. Ensure statistical functions match NumPy defaults (e.g., `torch.std(unbiased=False)`).
