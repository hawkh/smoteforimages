## 2024-05-15 - Optimize Distance Computation in QualityAssessor
**Learning:** In `QualityAssessor.compute_diversity_metrics`, pairwise distances were calculated by moving GPU tensors to the CPU via `.cpu().numpy()` and utilizing `sklearn.metrics.pairwise.pairwise_distances`. This is extremely inefficient because it introduces host-device memory transfers and bypasses potential GPU acceleration. PyTorch provides a native equivalent, `torch.cdist`, which avoids these transfers and runs directly on the tensor's device. Also, when creating index tensors (like with `torch.triu_indices`), they default to CPU, so passing the device explicitly (e.g., `device=tensor.device`) is crucial for optimal performance.
**Action:** When computing pairwise operations between PyTorch tensors, always prefer PyTorch native functions (`torch.cdist`, `torch.nn.functional.pdist`) over external libraries like NumPy/scikit-learn. Explicitly define the `device` parameter for tensor constructors (like index arrays) to match the target device.
